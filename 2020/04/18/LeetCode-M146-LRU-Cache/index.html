LeetCode M146 LRU Cache

发表于 2020-04-18 | 更新于 2020-04-19 | 课程 Algorithms | 阅读次数：

Description

Design and implement a data structure for Least Recently Used (LRU) cache. It should support the following operations: get and put.

get(key) - Get the value (will always be positive) of the key if the key exists in the cache, otherwise return -1.

put(key, value) - Set or insert the value if the key is not already present. When the cache reached its capacity, it should invalidate the least recently used item before inserting a new item.

The cache is initialized with a positive capacity.

Follow up:
Could you do both operations in O(1) time complexity?

Example:

LRUCache cache = new LRUCache( 2 /* capacity */ );

cache.put(1, 1);
cache.put(2, 2);
cache.get(1);       // returns 1
cache.put(3, 3);    // evicts key 2
cache.get(2);       // returns -1 (not found)
cache.put(4, 4);    // evicts key 1
cache.get(1);       // returns -1 (not found)
cache.get(3);       // returns 3
cache.get(4);       // returns 4

Thinking

It’s hard for me to figure it out what is LRU cache by the given example but not to read the explanation in the hyperlink linked to wiki. So i try to solve this problem by using queue and define the class Node store key and value. Certainly, i failed.

The LRU discards the least recently used items first. The use including get and put.

  • So, when put new node, we should remove the last node which is not used for the longest time if the cache is full. Aha! It sounds like remove tail node from list.
  • When we use one node, it should be placed in the first place.

Due to the above needs, double linked list is a good choice.

The most voted post in disscuss says:

One interesting property about double linked list is that the node can remove itself without other reference. In addition, it takes constant time to add and remove nodes from the head or tail.
to find the node by key
The problem can be solved with a hashtable that keeps track of the keys and its values in the double linked list.
improve by use pseudo head and tail
One particularity about the double linked list that I implemented is that I create a pseudo head and tail to mark the boundary, so that we don’t need to check the NULL node during the update. This makes the code more concise and clean, and also it is good for the performance.
