ConcurrentLinkedDeque in Java

发表于 2021-03-18 | 课程 Java

An deque implemented by linked transient volatile nodes and AQS.

Learn From Doc

  1. An unbounded concurrent deque based on linked nodes. Actually, it’s max size is Integer.MAX_VALUE.
  2. Iterators and spliterators(used to iterate elements concurrently) are weakly consistent rather than fast-fail.
  3. size() is not constant-time operation.Because of the asynchronous nature of these deques it will traverse the elements.
  4. Bulk operations that add, remove, or examine multiple elements are not guaranteed to be performed atomically.
  5. actions in a thread prior to placing an object happen-before actions subsequent to the access or removal of that element.

ArrayDeque in Java

发表于 2021-03-18 | 更新于 2021-03-19 | 课程 Java

An deque implemented by array and head, tail index.

Learn From Doc

  1. grow as necessary to support usage.
  2. not thread-safe, in other words, not support concurrent access by multiple threads.
  3. Null elements are prohibited.
  4. faster than stack and linkedList used as queue.
  5. most methods run in O(1).
  6. fail-fast iterator: only the iterator’s remove method can modify the deque after the iterator is created.
