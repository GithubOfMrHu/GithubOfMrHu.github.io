Shallow Copy and Deep Copy in Java

发表于 2020-04-07 | 更新于 2020-04-09 | 课程 Java | 阅读次数：

Object Copy

Java中的对象拷贝(Object Copy)指的是将一个对象的所有属性（成员变量）拷贝到另一个有着相同类类型的对象中去。

在程序中拷贝对象是很常见的，主要是为了在新的上下文环境中复用现有对象的部分或全部数据。

Java中的对象拷贝主要分为：浅拷贝(Shallow Copy)、深拷贝(Deep Copy)。

Shallow Copy

浅拷贝（Shallow Copy）只复制当前的对象，但不复制该对象内部的引用（在堆内存中从新创建空间，内容相同，但地址不同）。
Java中的数据类型分为基本数据类型和引用数据类型。在用进行赋值操作(=)、用作方法参数或返回值时会进行浅拷贝：

  • 对于基本数据类型（boolean, byte, short, char, int, float, double, long 8 种），进行的是值传递。也就是将该属性值复制一份给新的对象。因为是两份不同的数据，所以对其中一个对象的该成员变量值进行修改，不会影响另一个对象拷贝得到的数据。
  • 对于引用数据类型，是引用（地址）传递。比如说某个数组、某个类的对象等，那么浅拷贝会进行引用传递，也就是只是将该成员变量的引用值（内存地址）复制一份给新的对象。因为实际上两个对象的该成员变量都指向同一个实例。在这种情况下，在一个对象中修改该成员变量会影响到另一个对象的该成员变量值。

    Deep Copy

    对对象内部所有对象的引用均复制，每个引用对象创建一个实例并复制实例。

Dynamic Programming

发表于 2020-04-03 | 更新于 2020-04-07 | 课程 Algorithms | 阅读次数：

Explanation

动态规划（Dynamic Programming）与分治方法相似，都是通过组合子问题的解来求解原问题（这里的Programming指的是一种表格法，并非编写计算机语言）。

Difference Between Divide and Conquer

区别在于分治法将问题划分为互不相交的子问题，递归的求解子问题，再讲问题的解组合起来得到问题的解。而动态规划应用于子问题重叠的情况，即不同的子问题具有公共的子子问题。此时使用分治法时会反复求解那些公共子子问题。而动态规划通过表格记录对每个子子问题只计算一次，避免了不必要的计算工作。

Application

动态规划通常用来解决最优化问题（optimization problem）。注意，此类问题的最优解即具有最优值的解可能有多个，我们称这样的解为这个问题的一个最优解（an optinal solution），而不是最优解(the optional solution).

4 Steps to Design a Danamic Programing Solution

通常按以下四个步骤来设计一个动态规划算法：

  1. 刻画一个最优解的结构特征。
  2. 递归的定义最优解的值。
  3. 计算最优解的值，通常采取自底向上的方式。
  4. 利用计算出的信息构造一个最优解。
    步骤1~3是动态规划算法求解问题的基础，如果只需要最优解的值，而非解本身，可以忽略步骤4。
